# webtest

## 1. 프로젝트 개요 (Overview)

이 프로젝트는 DLP 보안솔루션의 관리자 웹 UI 테스트 및 기타 테스트를 담당합니다.

## 2. 왜 이 프로젝트가 필요한가? (Problem)

DLP UI 에는 수많은 요소가 있습니다. 단, 하나의 요소가 변경되면 나머지 요소도 영향을 받을 수밖에 없습니다. 
이 프로젝트는 새로운 패키지가 나왔을 때 기본이자 필수 기능인 '로깅' 및 '차단' 기능의 설정을 자동화한 것입니다. 
기존 짧게는 1시간 길게는 2시간 소요되던 웹설정을 이 자동화로 인해 5분만에 완료합니다.
ㄷ
## 3. 해결 방법 (Solution

selenium 코드를 활용하여 웹의 필수 설정 항목들을 자동으로 설정합니다. 

## 4. 사용 기술 (Tech Stack)

- Language : Python 
- Framework : Selenium 
## 5. 실행 방법 (How To Run)

해당 프로젝트에 포함된 크롬 드라이버와 사용자의 크롬 버전이 일치하는지 확인 후 일치하지 않으면 사용자 버전에 맞춘 크롬 드라이버를 설치 적용합니다. 
이후 dlptest/webset/init.py 에서 주석처리를 확인한 뒤 실행합니다.

## 6. 한계와 개선 방향 (Limitations & Future Work)

10.0.5 버전이 나온 이후 일부 UI 요소가 변경되었습니다. 해당 요소에 대한 유지보수는 이루어지지 않은 상태입니다. 
변경된 UI 요소 설정을 수정해야 하고, 이후에는 단순히 설정하는 것뿐아니라 UI 요소 테스트를 위해 활성화->비활성화->팝업 등을 확인하여 pass/fail 을 판별하도록 고도화해야 합니다.

## 7. 이 프로젝트를 통해 얻은 것

UI 요소가 브라우저, 개발 코드 등 수많은 요소에 의해 영향받는 것을 이해했습니다. 
요소의 DOM 이 완전히 로드되지 않으면 선택되지 않거나, 외부의 변경이 가해지면 그대로 중지되는 단점이 있습니다. 
selenium 의 여러 단점을 보완한 것이 playwright 이고, 서비스 자동화는 playwright으로 구현되었습니다. 
제작자가 처음 자동화 테스트를 공부하게 됐고, 구현하게 된 프로젝트입니다.
